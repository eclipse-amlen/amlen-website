Cleaning the store by using the Amlen WebUI

System administrators can clean the store by using the Server Control page in the Amlen WebUI. Cleaning the store deletes all persistent data.

If you are cleaning the store in an HA environment, you must follow the procedure and complete each step on both Eclipse Amlen servers before proceeding to the next step. For example, complete step 1 on the primary server and on the standby server before you begin step 2.

CAUTION: Cleaning the store deletes all persistent data, including messages, subscriptions, and client states. Do not clean the store unless you want to delete all persistent data on the system.
  1. Go to the Server Control page by selecting Server Control from the Server menu.
  2. Check the status of the server in the Status field of the IBM Watson IoT Platform - Message Gateway server pane:
    • If the server status is Running, the server is running in production mode.
    • If the server status is Maintenance, the server is in maintenance mode.
  3. Optional: If the server state is Stopped, you must start the server before you can continue. For more information about starting the server, see Starting, restarting and stopping the Eclipse Amlen server by using REST Administration APIs.
  4. Stop the MQ Connectivity service by clicking Stop the service in the MQ Connectivity Service pane.
  5. Optional: If the server state is Running, you must set the server state to Maintenance mode by clicking Start maintenance in the IBM Watson IoT Platform - Message Gateway Server pane. You must click Restart for the changes to take effect.
  6. Clean the store:
    1. Click Clean store in the IBM Watson IoT Platform - Message Gateway Server pane.
    2. Click Clean store and restart.
  7. Change the run mode from Maintenance to Running by clicking Stop maintenance in the IBM Watson IoT Platform - Message Gateway Server pane. You must click Restart for the changes to take effect.
