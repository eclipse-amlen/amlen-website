LDAP is not working

If the LDAP certificate check fails when you are creating an LDAP configuration object, you might need to enable DNS.

LDAP certificate check fails when you are creating an LDAP connection by using ldaps://

When you are creating an LDAP connection and specifying a URL that starts with ldaps://, you might see the following message:

Failed to bind with the LDAP server: Can't contact LDAP server. The error code is -1.

If you see this message, ensure that you enable DNS by adding a DNS entry to resolve the IP address of the LDAP server.
