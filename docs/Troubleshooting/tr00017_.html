Changing the server state by using the Amlen WebUI

System administrators can change the server state from Maintenance to Running, or from Running to Maintenance, by using the Server Control page in the Amlen WebUI.

For more information about maintenance mode, see Maintenance mode.

You change the server state from Maintenance to Running, or from Running to Maintenance, by using the Amlen WebUI, or by using REST Administration APIs. For more information about using REST Administration APIs to change the server state, see Changing the server state by using REST Administration APIs.

  1. Go to the Server Control page by selecting Server Control from the Server menu.
  2. Check the status of the server in the Status field of the IBM Watson IoT Platform - Message Gateway Server pane:
    • If the server status is Running, the server is running in production mode.
    • If the server status is Maintenance, the server is in maintenance mode.
  3. Choose an action:
    • Change the server state from Running, to Maintenance mode by clicking Start maintenance.
    • Change the server state from Maintenance mode, to Running by clicking Stop maintenance.
    • Restart the server by clicking Restart the server.
    • Stop the server by clicking Stop the server.
      Note: If you stop the server, you can only restart the server by using a docker or systemctl command. You cannot restart the server by using the Amlen WebUI or by using REST Administration APIs.
