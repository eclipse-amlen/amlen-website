Upgrading and rolling back the software level of Eclipse Amlen

System administrators can upgrade the software level of the Eclipse Amlen by applying interim fixes. The software level can also be rolled back by removing interim fixes.

You can upgrade or roll back the software level of your Eclipse Amlen that is running in a Docker container or as an RPM on the host OS by applying and removing interim fixes.

Install the update level that you require when you first install Eclipse Amlen. The format of the images that are provided as updates for Eclipse Amlen are identical to the GA-level installation image; there is no need to install the GA-level image before you apply updates.

You can find the list of interim fixes at IBM Support

  • Upgrading Eclipse Amlen in a Docker environment
    System administrators can upgrade the Eclipse Amlen server or Web UI running in a Docker container.
  • Rolling back the level of Eclipse Amlen in a Docker environment
    If, after an upgrade of the software level of Eclipse Amlen in a Docker environment, a rollback to a prior level is required, system administrators can perform the rollback operation.
  • Upgrading the Eclipse Amlen RPM
    System administrators can upgrade the Eclipse Amlen server or Web UI running as RPMs on the host OS.
  • Rolling back the Eclipse Amlen RPM
    If, after an upgrade of the software level of the Eclipse Amlen RPMs, a rollback to a prior level is required, system administrators can perform the rollback operation.
