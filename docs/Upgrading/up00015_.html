Upgrading from IBM IoT MessageSight version 1.2

System administrators can upgrade to Eclipse Amlen version 0.1 from IBMÂ® IoT MessageSight version 1.2.

System administrators can upgrade their IBM IoT MessageSight version 1.2 systems to Eclipse Amlen version 0.1 in one of two ways. Both forms of upgrade involve migrating data from an IBM IoT MessageSight version 1.2 physical or virtual appliance to a running Eclipse Amlen version 0.1 environment.
  • Configuration migration. Only configuration items from the server such as endpoints, messaging policies, connection policies, LDAP configuration objects, and any configuration objects that are configured by using commands by an administrator are included in the upgrade process. The server store is not migrated.
  • Store migration in a high availability environment. The server configuration and the contents of the store are included in the upgrade process. The store contains buffered messages, retained messages, subscriptions, and topics. It contains all persistent messages and objects that have been dynamically created by clients.
  • Planning to upgrade to Eclipse Amlen version 0.1 from IBM IoT MessageSight version 1.2
    System administrators can select one of two methods to upgrade IBM IoT MessageSight version 1.2 to Eclipse Amlen version 0.1. Compare the two methods and understand the advantages and disadvantages of each method.
  • Upgrading to Eclipse Amlen version 0.1: server configuration only
    System administrators can upgrade their IBM IoT MessageSight version 1.2 systems to Eclipse Amlen version 0.1 by exporting and importing the server configuration file. The server configuration file is exported from an IBM IoT MessageSight version 1.2 physical or virtual appliance and imported into a running Eclipse Amlen version 0.1 environment.
  • Upgrading to Eclipse Amlen version 0.1: server configuration and store
    System administrators can upgrade their IBM IoT MessageSight version 1.2 systems to an Eclipse Amlen version 0.1 environment by migrating the configuration and store from an IBM IoT MessageSight version 1.2 physical or virtual appliance to a running Eclipse Amlen version 0.1 environment. The upgrade process requires the use of high availability (HA) servers.
  • Upgrading to Eclipse Amlen version 0.1: taking a backup of the IBM IoT MessageSight version 1.2 configuration and store
    Before you migrate the IBM IoT MessageSight version 1.2 configuration file and store to a running Eclipse Amlen version 0.1 server, it is best practice to take a backup of the production server store and copy it to a test environment to test the upgrade process.
