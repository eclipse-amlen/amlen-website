MQ Connectivity scenarios

The MQ Connectivity scenarios extend the basic Eclipse Amlen scenarios by replacing part of the solution with IBMÂ® MQ topics or queues.

For more information about the basic Eclipse Amlen scenarios, see End-to-end scenarios.

  • MQ Connectivity scenario 1: Fan out broadcast
    In this scenario, one IBM MQ queue is mapped to an Eclipse Amlen topic. Messages that are put on the IBM MQ queue are broadcast to a number of MQTT application subscribers.
  • MQ Connectivity scenario 2: Fan in per device notification
    In this scenario, one Eclipse Amlen topic is mapped to a IBM MQ queue. Messages that are published to the Eclipse Amlen topic are forwarded to the IBM MQ queue.
  • MQ Connectivity scenario 3: Fan out per device notification
    In this scenario, a IBM MQ topic subtree is mapped to an Eclipse Amlen topic subtree. Messages that are put on topics in the IBM MQ subtree are forwarded to the equivalent Eclipse Amlen topic subtree.
  • MQ Connectivity scenario 4: Fan out per device request-reply
    In this scenario, a IBM MQ topic subtree is mapped to an Eclipse Amlen topic subtree for requests. Messages that are put on topics in the IBM MQ topic subtree are forwarded to the equivalent Eclipse Amlen topic subtree for requests. A different Eclipse Amlen topic subtree is mapped to a IBM MQ topic subtree for responses. Messages that are put on topics in the Eclipse Amlen topic subtree are forwarded to the equivalent IBM MQ topic subtree for responses.
  • MQ Connectivity scenario 5: Fan in per device request-reply
    In this scenario, an Eclipse Amlen topic subtree is mapped to a IBM MQ topic subtree for requests. The messages that are put on topics in the Eclipse Amlen topic subtree are forwarded to the equivalent IBM MQ topic subtree for requests. A different IBM MQ topic subtree is mapped to a Eclipse Amlen topic subtree for responses. Messages that are put on topics in the IBM MQ topic subtree are forwarded to the equivalent Eclipse Amlen topic subtree for responses.
