Configuring client certificates

You can authenticate based on client certificates by uploading the root CA certificates and intermediate CA certificates, or the client certificates to the Eclipse Amlen truststore. You can upload the CA certificates or client certificates by using the Amlen WebUI or by using REST Administration APIs.

Each CA certificate or client certificate that you upload is associated with a security profile. The certificate is used with that security profile to authenticate certificates that are presented by client applications. For performance reasons, you might want to upload CA certificates only. Uploading certificates for each client and associating each client certificate with a security profile might affect connection performance. For more information about client certificates and the truststore, see Client certificate authentication.

For more information about uploading a CA certificate or a client certificate by using the Amlen WebUI, see Configuring client certificates by using the Amlen WebUI.

For more information about uploading a CA certificate or a client certificate by using REST Administration APIs, see Creating and updating a client certificate by using REST Administration APIs.
