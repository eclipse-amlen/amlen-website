Removing an inactive cluster member by using the Amlen WebUI

System administrators can remove an inactive cluster member by using the Cluster menu options in the Amlen WebUI.

Only take this action if you have a cluster member that is inactive and cannot be restarted. Removal of an inactive cluster member results in that member being permanently removed from the cluster. All messages that are queued to be sent to the inactive member are permanently deleted from the system. Outbound messages from the inactive member are also permanently deleted. If you have a member that you no longer want to participate in the cluster, disable cluster membership by using the Cluster Membership page. For more information about disabling cluster membership, see Configuring cluster membership by using the Amlen WebUI.

When a cluster member is not available for an extended period or is inactive, you can remove that cluster member from the discovery list of all active cluster members to avoid a potential impact on performance. Only use this REST API if a cluster member cannot be recovered. In all other instances, disable cluster membership on the active member that you want to remove.

You can query cluster members with a status of inactive by using Cluster monitoring.

To remove an inactive cluster member, complete the following steps:
  1. Go to the Cluster Status page by selecting Cluster > Status.
  2. Select the server that you want to remove from the Cluster Status Data table.
  3. Click Delete to remove the inactive cluster member.
