Scenarios for configuring the Eclipse Amlen resource adapter for inbound communication

There are some basic, underlying tasks that are common to each of the following scenarios that are associated with configuring the Eclipse Amlen resource adapter for inbound communication. The details for each task vary according to requirements. However, the following steps must be considered, understood, and completed to configure the resource adapter for inbound communication.

  1. Download and install the Eclipse Amlen resource adapter.
  2. Configure an endpoint and setup connection and messaging policy types.
  3. Configure Eclipse Amlen resource adapter properties.
  4. Deploy and start your MDB if applicable.

The following topics provide more detailed information about parameters that you might want to consider when you configure the Eclipse Amlen resource adapter for different scenarios.

  • Configuring Eclipse Amlen resource adapter in a WebSphere Application Server cluster
    The rapid growth of mobile devices is resulting in an increase in the volume of data that flows into an enterprise. To manage the volume of data, the Eclipse Amlen resource adapter can be installed and configured to support horizontal scaling across a WebSphere® Application Server cluster.
  • Configuring the Eclipse Amlen resource adapter to preserve message order
    Message order is important to some asynchronous messaging applications where messages must be processed in the same order in which the producer sends them. Your design must take message order into account if message order is important to your application.
  • Configuring Eclipse Amlen resource adapter for scaling within a single WebSphere Application Server
    In this configuration, the number of concurrent consumers are increased within a single WebSphere® Application Server. Scaling the number of consumers enables messages to be processed concurrently. If messages are processed concurrently, then message order cannot be preserved.
  • Configuring Eclipse Amlen resource adapter for high availability
    Eclipse Amlen might experience planned and unplanned outages. High availability (HA) provides the ability for messaging services to withstand outages and continue providing processing capability.
