Creating and updating a messaging policy type by using REST Administration APIs

You can use the Eclipse Amlen REST API POST method to create or update one or more types of messaging policy object. There are three types of messaging policy; topic policy, subscription policy and queue policy. If the type of messaging policy that is specified in the request does not exist in the Eclipse Amlen configuration, then the policy is created. Otherwise, the existing policy configuration is updated.

Use one or more of the following links to create the type of messaging policy that you require:

  • Create a topic policy.

  • Create a subscription policy.

  • Create a queue policy.
