Starting, restarting, stopping, and viewing the status of MQ Connectivity

You can start, restart and stop the MQ Connectivity process by using REST Administration APIs. You can restart and stop the MQ Connectivity process by using the Amlen WebUI.

The status of the MQ Connectivity process determines what actions can be carried out. You can view the MQ Connectivity process status by using a REST Administration API, or by using the Amlen WebUI.

  • To start the MQ Connectivity process, MQConnectivity must be enabled or the process will fail to start. Setting MQConnectivityEnabled to true automatically starts the MQ Connectivity process.

  • If the MQ Connectivity process status is Inactive, and MQConnectivityEnabled is true you can start or restart the MQ Connectivity process.
  • If the MQ Connectivity process status is Active, the MQ Connectivity process is running. You can stop the MQ Connectivity process.

For more information about viewing the status of the MQ Connectivity process, and restarting and stopping the MQ Connectivity process by using the Amlen WebUI, see Restarting, stopping, and viewing the status of MQ Connectivity by using the Amlen WebUI.

For more information about viewing the status of the MQ Connectivity process by using REST Administration APIs, see Viewing the status of the MQ Connectivity process by using a REST Administration API.

For more information about starting, restarting and stopping the MQ Connectivity process by using REST Administration APIs, see Starting, restarting and stopping the MQ Connectivity process by using REST Administration APIs.
