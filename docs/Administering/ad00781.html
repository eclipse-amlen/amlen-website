Changing the state of an XA transaction branch by using REST Administration APIs

System administrators can commit an XA transaction branch that is in prepared state by using REST Administration APIs. System administrators can roll an XA transaction branch back that is in prepared state. System administrators can forget an XA transaction branch that is in a heuristically complete state.

You can list the active XA transactions to get the required transaction IDs (XIDs) by using XA Transaction monitoring. For more information about XA Transaction monitoring, see XA Transaction monitoring.
  • Committing an XA transaction branch by using REST Administration APIs
    System administrators can heuristically commit an XA transaction branch that is in prepared state by using REST Administration APIs.
  • Rolling back an XA transaction branch by using REST Administration APIs
    System administrators can heuristically roll back an XA transaction branch that is in prepared state by using REST Administration APIs.
  • Forgetting an XA transaction branch by using REST Administration APIs
    System administrators can forget a heuristically completed XA transaction branch by using REST Administration APIs.
