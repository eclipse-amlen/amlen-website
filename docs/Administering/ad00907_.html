Bring back both appliances after both appliances fail simultaneously

In this scenario, a massive power loss causes both the primary node and the standby node to fail. You must bring both nodes back up.

Use the following information to understand the steps that are required to bring back the original primary node (Appliance A) as primary, and the original standby node (Appliance B) as standby after both node fail simultaneously. The content of the store and the configuration of Appliance A is preserved and Appliance A and B require a restart. Appliance A remains as primary as that appliance has the most recent data.



You can configure appliances for high availability by using the Amlen WebUI, or by using the command line. For more information about using the Amlen WebUI to configure appliances for high availability, see Bring back both appliances after both appliances fail simultaneously by using the GUI. For more information about using the command line to configure appliances for high availability, see Bring back both appliances after both appliances fail simultaneously by using the command line.
