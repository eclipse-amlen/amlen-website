Administering

Administration tasks can be completed by system administrators or messaging administrators by using the Amlen WebUI, or by using REST Administration APIs.

Administering by using the web UI

You can connect to the Amlen WebUI by using a web browser. For more information about how you can connect to the Amlen WebUI, see Logging in to the Amlen WebUI

Administering by using REST Administration APIs

Eclipse Amlen has a REST (REpresentational State Transfer) interface and provides REST Administration APIs which you can use to administer and monitor your system.

To run a REST Administration API, call a method on a REST resource and pass parameters or a request in JSON format. The methods that the REST Administration APIs use are POST, GET, and DELETE:
  • The POST method creates or updates a configuration object.
  • The GET method retrieves data about a configuration object.
  • The DELETE method deletes a configuration object.
You can write scripts or use the Linux™ program cURL to run the REST Administration APIs. Combine a REST URI with the REST method to be used and the data to be passed to create the REST request.
  • Configuring the administration endpoint
    You can configure the administration (admin) endpoint either by using the Amlen WebUI or by using the REST Administration API.
  • Server administration
    System administrators can administer the server by using the Amlen WebUI, or by using REST Administration APIs.
  • Configuring LDAP for user authentication
    You can configure messaging users and groups by using an LDAP server. You can configure the LDAP server either by using the Amlen WebUI, or by using REST Administration APIs.
  • Configuring security
    You can configure a number of security components. You can configure certificate profiles and security profiles to secure your endpoints. You can upload client certificates to the Eclipse Amlen truststore. You can configure Lightweight Third Party Authentication (LTPA) and OAuth. You can enable FIPS-compliant security. All these security components can be configured either by using the Amlen WebUI or by using REST Administration APIs.
  • Configuring protocol plug-ins
    The protocol plug-in provides the ability to extend the protocols that are supported by the Eclipse Amlen server. You can configure protocol plug-ins by using REST Administration APIs.
  • Configuring message hubs
    You can configure messaging hubs either by using the Amlen WebUI or by using REST Administration APIs.
  • Configuring message queues
    You can configure queues either by using the Amlen WebUI or by using REST Administration APIs.
  • Configuring the cluster membership of an Eclipse Amlen server
    You can configure Eclipse Amlen servers in clusters to increase the maximum messaging throughput or the maximum number of concurrent connections beyond the capacity of a single Eclipse Amlen server. All Eclipse Amlen servers in a cluster share a common topic tree.
  • Configuring disconnected client notifications
    To configure disconnected client notifications, you must configure Eclipse Amlen and a notification provider. You can use notification providers such as Worklight®, or you can create your own MQTT application to handle the notifications.
  • Configuring administrative subscriptions
    Use administrative subscriptions to ensure that a subscription continues to exist and receive messages when there are no clients that are using the subscription.
  • Deleting a ClientSet by using a REST Administration API
    System administrators can delete a ClientSet by using the Eclipse Amlen REST API DELETE method.
  • Configuring SNMP
    System administrators can configure Eclipse Amlen to use the Simple Network Management Protocol (SNMP) by using the Amlen WebUI, or by using REST Administration APIs.
  • Configuring certificate revocation lists
    When using client certificates to control access for client devices, if you have a device which has been compromised you can blacklist that device by using a Certificate Revocation List.
  • Viewing and deleting configuration objects by using REST Administration APIs
    System administrators can view the configuration details of configuration objects and can delete configuration objects by using REST Administration APIs or the Amlen WebUI. For information about using the Amlen WebUI to view details of a configuration object and to delete a configuration object, see the appropriate information that relates to using the Amlen WebUI to configure the configuration object.
  • Managing multiple Eclipse Amlen servers from a single Amlen WebUI
    You can use a single Amlen WebUI instance to manage and monitor multiple Eclipse Amlen servers that are version 2.0 or higher. The first time that you log into the Amlen WebUI, you are prompted for information about the first Eclipse Amlen server that you would like to manage. After your first login, you can add, edit or remove Eclipse Amlen servers to your managed server list by using the Server menu in the Amlen WebUI.
  • Working with external resources
    You can configure Eclipse Amlen to work with external resources by using MQ Connectivity or the Eclipse Amlen JMS resource adapter.
  • HTTP status codes
    You can use standard HTTP status codes to understand the success or failure of a request.
