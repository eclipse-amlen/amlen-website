High availability scenarios

The following scenarios assume that you are configuring your servers to run as an HA pair automatically, by enabling High Availability and by using the High Availability Group setting.

  • Add a new standby node after the primary node failed
    In this scenario, the primary node of an HA pair fails and the standby node takes over. You must now set up a new standby node for the new primary node.
  • Add a new standby node after the original standby node failed
    In this scenario, the standby node of an HA pair has failed. So, you must add a new standby node.
  • Bring back a standby node after it failed
    In this scenario, the standby node of an HA pair failed. After you have fixed the problems with the standby node, you must bring it back up as a backup.
  • Bring back primary and standby nodes after both nodes fail
    In this scenario, the primary node has failed and the standby node has taken over as the new primary node. Before a new backup can be brought up, the new primary node fails. You need to bring back both nodes.
  • Bring back the last primary node after the primary and then the standby node fails
    In this scenario, the primary node fails and the standby node takes over as the new primary node. Before a new backup can be setup, the new primary node fails. You must restart new primary node and set up a new standby node.
  • Bring back both appliances after both appliances fail simultaneously
    In this scenario, a massive power loss causes both the primary node and the standby node to fail. You must bring both nodes back up.
