Viewing disconnected MQTT client-related statistics by using the Amlen WebUI

Use the disconnected MQTT clients monitor to view unconnected MQTT client statistics.

You can view unconnected MQTT client monitoring statistics by using the Amlen WebUI, or by using a REST Monitoring API. For more information about using a REST Monitoring API to view inactive MQTT client monitoring statistics, see Viewing disconnected MQTT client-related statistics by using REST Monitoring APIs. For more information about the predefined queries that you can run, see MQTT Client monitoring.

  1. From the Amlen WebUI, select Monitoring > Disconnected MQTT Clients.
  2. Enter a value into Client ID. The default value is *.
  3. Click Refresh to generate the output in the MQTT clients table.
  4. You can limit the number of clients that are added to the table by selecting an value of 10, 25, 50, or 100. The default value is 10.
  5. Enter a value into Filter to change the visibility of the data that is shown in the table. The filter is applied to the maximum of items that are returned by the query.
  6. To Delete a client (and any subscriptions that are associated with the client), complete the following steps:
    1. From the MQTT client table, select the client that you want to delete.
    2. Click Actions, and select Delete Client to delete the client, and any subscriptions that are associated with the client.

The Timestamp is updated to show the time that query is generated.

The MQTT Clients table is updated to display the results of the query and any filter that is applied to those results.

You can view the total number of items that are returned in the Total field. You can see the number of items that are selected in the Selected field. You can move through the table of results by using Next Page and Previous Page or Go to a specific page. You can change the number of items to view per page by selecting from the available options.
