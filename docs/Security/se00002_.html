Messaging users and groups

System administrators can configure Eclipse Amlen to access an LDAP server to authenticate messaging users and groups. Messaging users and groups can be authorized by system and messaging administrators to publish, and subscribe to topics.

Each messaging user has a unique messaging user ID. This messaging user ID can be used in a connection policy to allow or deny the user a connection to Eclipse Amlen. The messaging user ID can be used in a messaging policy to control whether the user can publish or subscribe to a topic. The messaging user ID can also be used in a messaging policy to control whether the user can send, receive, or browse a queue.

Messaging users can be part of messaging groups. Adding users to a group can simplify administration tasks. For example, you might want one set of users to publish to a particular topic, and another set of users to subscribe to a particular topic. Instead of creating individual messaging policies for each user, you can create a messaging group who are authorized to publish to a topic. You create another messaging group authorized to subscribe to a topic. Users can be added to the relevant messaging group, and that group can then be associated with the appropriate messaging policy.

For more information about configuring LDAP for use in user authentication, see Configuring messaging users and groups.
