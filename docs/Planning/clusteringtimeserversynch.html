Clustering: Time server synchronization

Synchronize all servers in a cluster by using a common time server.

Servers in an Eclipse Amlen cluster need a consistent way of determining which messages are newer than which other messages, regardless of the server to which the application that publishes them is connected. This is particularly important when it comes to maintaining a consistent set of retained messages throughout the cluster; as messages arrive at servers, perhaps in different orders, it needs to be possible to identify which are newer retained messages than the retained messages that the server already knows.

For this reason, you must synchronize all the servers in a cluster by using a common time server.
